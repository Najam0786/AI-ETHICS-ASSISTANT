Article III — Mechanism

Like the Act itself,
built from discrete steps.

retrieve → generate → verify → revise → finalize

Retrieve

MMR search across 758 chunks — diverse context, not near-duplicates.

Generate

Groq drafts an answer strictly from retrieved context.

Verify

An independent second call checks citation, faithfulness & relevance.

Revise

Only if verification fails — regenerates once, told exactly why.

Remember

LangGraph MemorySaver keeps context across a conversation thread.

Amendment — on the record

The brief specified Google Gemini. Gemini's free tier caps embeddings at 1,000/day and 100/minute — for 758 chunks, that quota blocked development repeatedly.

We migrated to Groq + local embeddings — both genuinely free, no rate limits. The architecture, agent design, and prompt are unchanged; only the provider is.
